How to think metadata

what does it take to set up a catalogue that really serves the organization

Ask questions

  • Do you have different categories of data?
  • Are there different temporal and spatial subgroups?
  • Do you have data sets with multiple sources?
  • Is it necessary to refer to Online Resources?
  • What is needed to describe a series of related granules?
  • Is it necessary to describe types of aggregations?
  • Does the data quality vary in the data set?
  • What is needed to track multiple data sources
  • What is needed to track compliance with standards?
  • Is it necessary to use spatial characteristics to describe quality, such as quality indicators?
  • It is necessary to explain why procedures were performed on the data?
  • Do you have data sets in multiple locations?
  • Is it necessary to describe the instruments used to make observations?
  • Is it necessary to unequivocally identify the namespaces used?
    Do you want to serve metadata using a REST (Representational State Transfer) web service?
  • Is it necessary to track user problems?
  • What is needed to track the requirements and plans?
  • What is needed to track transformations and data processing?
